# Family Multiplayer Home Games

A self-learning project implementing a structured 7-stage framework for listing and optimizing multiplayer family games.

## What is this?

This repository organizes local multiplayer family games suited for kids and TV couch-play. It follows a disciplined 7-stage project architecture using AI agents (Claude, Gemini, Copilot, Kilo Code) as partners:

1. **Problem Discovery** — Define unknowns, OKRs, and core questions
2. **Environment Setup** — Establish context, constraints, and tooling
3. **Simulation** — Visualize the vision with mockups and diagrams
4. **Formula** — Research, plan, and document the step-by-step approach
5. **Symbols** — Write the actual code (game list files: `index.html`, `kids-3-9.html`, `barbie.html`)
6. **Semblance** — Capture errors, workarounds, and lessons learned
7. **Testing** — Validate everything against the original unknowns

## 🧠 Fitness as a Self-Learning Platform

The 7-stage structure is highly suited to self-learning because it mimics the cognitive steps of learning:

1. **[1_Real_Unknown](file:///Users/rifaterdemsahin/projects/multiplayer-home-games/1_Real_Unknown)** maps to **Active Ignorance** (recognizing what you do not know).
2. **[2_Environment](file:///Users/rifaterdemsahin/projects/multiplayer-home-games/2_Environment)** & **[3_Simulation](file:///Users/rifaterdemsahin/projects/multiplayer-home-games/3_Simulation)** map to **Mental Sandbox** (building context and visualizing the system).
3. **[4_Formula](file:///Users/rifaterdemsahin/projects/multiplayer-home-games/4_Formula)** maps to **Synthesis** (developing recipes, logic, and planning).
4. **[5_Symbols](file:///Users/rifaterdemsahin/projects/multiplayer-home-games/5_Symbols)** maps to **Execution** (translating theory to reality).
5. **[6_Semblance](file:///Users/rifaterdemsahin/projects/multiplayer-home-games/6_Semblance)** maps to **Feedback Loop** (documenting mistakes, debugging, active reflection).
6. **[7_Testing_Known](file:///Users/rifaterdemsahin/projects/multiplayer-home-games/7_Testing_Known)** maps to **Consolidation** (proving outcomes).

## How to use

1. Open `index.html` locally or visit the GitHub Pages URL to view the Project Dashboard.
2. Navigate to **Play Games** to browse the multiplayer game portals with controller requirements.
3. Read `agents.md` for agent coordination rules.
4. Explore each stage folder (`1_Real_Unknown/` through `7_Testing_Known/`) for details on problem analysis, setups, and learnings.
